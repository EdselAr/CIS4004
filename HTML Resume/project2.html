Memory Management Unit (MMU) Simulation

Project Description

This project focuses on the fundamental operating system concept of Paging. Written in C, the program simulates a Memory Management Unit (MMU) that translates Logical Addresses into Physical Addresses.

The screenshot above demonstrates the program's error handling capabilities. When the user inputs a logical address (999), the system calculates the specific Page (3) and Offset (231). As shown in the output, it correctly identifies that "Page 3 not in memory," simulating a page fault or invalid access.

Key Features & Skills

  • System Programming: Implemented low-level memory logic using C.
  • Bitwise Operations: Used arithmetic logic to separate an address into Page numbers and Offsets.
  • Error Handling: Validates the presence of a page in the physical frame table before attempting access.
